# 2026-04-05_002 — Obsidian-Lite Vault Viewer

## What changed
New route `/workspace/[slug]/vault` in Mission Control — a three-column Obsidian-style viewer for browsing all project markdown documentation.

**Files created:**
- `src/lib/vault/collections.ts` — collection config (knowledge, vault, changelog, ADR)
- `src/lib/vault/indexer.ts` — server-side file scanner, frontmatter parser, wikilink resolver, search, graph builder
- `src/lib/vault-store.ts` — Zustand store for vault UI state
- `src/app/api/vault/tree/route.ts` — file tree API
- `src/app/api/vault/file/route.ts` — file read API with path security
- `src/app/api/vault/search/route.ts` — full-text search API
- `src/app/api/vault/graph/route.ts` — graph data API (nodes + edges)
- `src/app/api/vault/index/route.ts` — manual re-index endpoint
- `src/app/workspace/[slug]/vault/page.tsx` — main vault page shell
- `src/components/vault/VaultFileTree.tsx` — collapsible file tree sidebar
- `src/components/vault/MarkdownViewer.tsx` — markdown renderer with wikilink support
- `src/components/vault/MetadataPanel.tsx` — backlinks, tags, properties panel
- `src/components/vault/VaultSearch.tsx` — search with debounced input
- `src/components/vault/TagFilter.tsx` — tag cloud filter
- `src/components/vault/GraphView.tsx` — d3-force knowledge graph overlay

**Files modified:**
- `src/components/Header.tsx` — added Vault nav link (purple)
- `src/app/globals.css` — added vault-prose CSS + highlight.js dark theme
- `package.json` — added react-markdown, remark-gfm, rehype-highlight, gray-matter, d3-force, d3-selection, d3-drag, d3-zoom

## Why
120+ markdown files across 4 collections (knowledge/, klaude-vault/, CHANGELOG/, ADR/) had no browsable UI. Developers could only search via business-brain MCP or read files directly. This provides a visual browser with cross-reference navigation.

## Stack
Next.js 14, React 18, react-markdown, remark-gfm, rehype-highlight, gray-matter, d3-force, d3-drag, d3-zoom, d3-selection, Zustand, Tailwind CSS

## Integrations
- Reads `~/Desktop/klaude-vault/` (external Obsidian vault) via path-security allowlist
- Reads repo-local `knowledge/`, `CHANGELOG/`, `ADR/` directories
- Reuses existing `path-security.ts` for traversal protection

## How to verify
1. `npm run build` in `apps/mission-control/` — passes
2. Navigate to `/workspace/default/vault`
3. File tree shows 4 collections: Knowledge (16), Vault (6), Changelog (94), ADR (14)
4. Click a file — renders markdown with syntax highlighting, frontmatter badges
5. Wikilinks render as clickable cyan links (test vault journal entries)
6. Search "stripe" — returns results across collections
7. Click Graph button — force-directed graph with colored collection nodes
8. Backlinks and outgoing links shown in metadata panel

## Known issues
- Graph zoom uses d3-zoom but scroll-to-zoom may conflict with page scroll on some browsers
- Tag filter currently requires a separate search API call to discover all tags (could be optimized with a dedicated tags endpoint)
- No live filesystem watching — use Refresh button to re-index after external file changes
